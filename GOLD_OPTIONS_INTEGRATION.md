## 0. Зачем это (и почему осторожно)

На **честной** per-asset IV (`dynamic_sigma`, σ = realized-vol×1.05) золото — лучший по
риск-доходности (avg **+13.16%/сделку**, sharpe **0.25** против ETH 0.06) И максимально
**декоррелировано** с криптой (Jaccard ETH↔XAUT = 0.21; 63% золотых сигналов приходят, когда
ETH молчит). То есть золото реально добавляет торговые дни, а не дублирует ETH.

**НО три риска, которые обязаны быть закрыты до денег:**
1. **Тренд-зависимость.** В выборке золото сделало +72% безоткатным буллом; edge почти весь в
   Put-стороне (P +28.4% / C +0.4%). Это частично захват тренда, не чистый vol-edge. Нужен тест
   на боковике/медвежьем окне золота (вне 2025-26 буллрана).
2. **Ликвидность.** На Bybit всего 362 золото-контракта; в бэктесте зашит спред 2%, реальный для
   тонкого золота может быть шире → съест край. На Derive XAUT-опционы мёртвые (IV=плейсхолдер) —
   **золото торгуемо ТОЛЬКО на Bybit**.
3. **σ-флор.** Честный edge посчитан при σ≈0.20-0.27 (clamp-флор 0.20 поверх реальной RV ~19%).
   Если реальная IV золота в спокойные периоды <20%, премия и edge ещё меньше.

## 1. Что технически нужно добавить (минимальные изменения)

Бот уже **символ-агностичен** на уровне данных (`backtest_data.fetch_klines(symbol,...)` берёт
любой символ; `XAUTUSDT` — linear-перп, USDT-settled, как ETH). Менять исполнение НЕ нужно.
Требуется параметризовать по инструменту следующее:

### 1.1. Per-instrument конфиг (новый слой)
Сейчас параметры захардкожены под ETH. Ввести таблицу инструментов, напр. в
`backend/services/instruments.py`:

| Параметр | ETH (сейчас) | XAUT (золото) | где сейчас зашито |
|---|---|---|---|
| `symbol` | `ETHUSDT` | `XAUTUSDT` | — |
| `strike_step` | `25` | подобрать (см. ниже) | `backtest.py:119` `round(spot/25)*25` |
| `lot_eth` / min qty | `0.1` | по спецификации Bybit XAUT-опциона | `paper_strategy.LOT_MIN_ETH`, `live_sizing.LOT_ETH` |
| `im_rate` | `0.10` | уточнить у Bybit для золота | `paper_strategy.IM_RATE` |
| `iv_rv_multiplier` | `1.05` | калибровать к реальной золото-IV | `backtest.py` dynamic_sigma |
| `sigma_clamp` | `(0.20,1.50)` | напр. `(0.10,0.50)` — у золота vol ниже | `backtest.py` |
| exits (tp/sl/hold) | `PUT_EXIT/CALL_EXIT` | **переоптимизировать** на золоте | `strategy_config.py` |

> ⚠️ **Strike-баг.** `round(spot/25)*25` валиден для ETH/золота (~$4300), но НЕ универсален
> (для XRP ~$2 даёт страйк 0). При вводе таблицы сразу заменить на `round(spot/step)*step` с
> per-instrument `step`, иначе грабли на дешёвых монетах. Реальный шаг страйка XAUT-опционов
> взять из `instruments-info?category=option&baseCoin=XAUT` (поле `priceFilter`/strike spacing).

### 1.2. σ-калибровка под золото
Главный фикс достоверности. Реальная IV золота ≈ 15-20% (НЕ 60% крипто). Источник реальной IV:
**не Derive** (мёртвые контракты) — взять с CME gold options / GLD options, либо консервативно
использовать саму realized-vol золота (×~1.05). Понизить `sigma_clamp` флор до ~0.10.

### 1.3. Мульти-инструментный раннер
- `paper_loop` / live-loop сейчас крутит один символ. Вариант A (просто): отдельный сервис-контейнер
  `paper-xaut` с тем же кодом и `SYMBOL=XAUTUSDT` (как `trader`-профиль). Вариант B: внутренний цикл
  по списку инструментов. **Рекомендация — A** (изоляция, отдельный гейт, не трогаем рабочий ETH-бот).
- `MAX_OPEN_POSITIONS` и портфельная маржа должны считаться **на уровне аккаунта** (общий депозит),
  если золото и ETH делят один кошелёк → вынести cap в общий риск-слой.

### 1.4. Данные/скоринг для дашборда
- Гейдж близости-входа (`entry_proximity`) и `/paper/conditions` — параметризовать символом.
- Фронт: добавить переключатель инструмента или второй блок.

## 2. Backtest-gate перед деплоем (обязательно, по порядку)

1. **Переоптимизировать exits под золото** (его vol-режим иной) — харнесс `variant_backtest.py` с
   золото-данными; сравнить с дефолтными ETH-exits.
2. **Честная σ из реальной золото-IV** (не клампленная крипто-RV) — перепроверить, держится ли +edge.
3. **OOS / другой режим**: train/holdout по времени + прогон на куске золота вне буллрана. Edge должен
   пережить смену режима, иначе это трендовый артефакт → отказ.
4. **Ликвидность вживую**: снять реальный orderbook/спред XAUT-опционов (`get_orderbook category=option`),
   подставить фактический спред в бэктест вместо 2%.
5. **Paper-гейт** как у ETH: ≥20-30 циклов, paper в пределах 30-50% бэктеста, наблюдать SL/CB/dynsize.

## 3. Чего НЕ делаем
- НЕ деплоим золото в live до прохождения гейта §2 (особенно п.3 — тест вне буллрана).
- НЕ трогаем рабочий ETH-paper-бот (золото — отдельный сервис/гейт).
- НЕ используем Derive для золота (контракты неликвидны). Только Bybit.
- BTC/SOL/XRP — отвергнуты на честной IV, в план не входят.
